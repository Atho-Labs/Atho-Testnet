## Why These Materials Are Kept

They are useful for:

- tracking original design intent
- comparing the current implementation against the rebuild plan
- understanding where certain subsystem priorities came from

They are not authoritative for:

- current runtime behavior
- current production-readiness
- current activation rules

Use the live docs and code for those topics.
